API & Integration

API overview, authentication, endpoints, and service integrations

API Overview

The system exposes a REST API built with Laravel. All API responses use JSON format. The API uses session-based authentication with role-based access control middleware.

Base URL

http://barangay-events.test

Protocol

HTTP/HTTPS

Format

JSON (application/json)

Authentication

Session-based with CSRF token + role middleware

Stack

Laravel 13.6 + Inertia.js (server-driven SPA)

Authentication

  • Session Authentication — Laravel built-in session guard for web routes
  • Role Middleware — Custom CheckRole middleware validates user role on each request
  • MustChangePassword Middleware — forces new users to set a password on first login
  • CSRF Protection — all state-changing requests require CSRF token
  • Webhook Signature — Stripe webhook requests are verified using STRIPE_WEBHOOK_SECRET

Integrations

Stripe

  • Handles paid event registration payments
  • Uses webhooks to auto-confirm registrations
  • Webhook endpoint: /api/webhooks/stripe

Pusher

  • Provides real-time updates in the UI
  • Broadcasts live slot availability changes
  • Pattern: Laravel Broadcasting to Vue frontend

Supabase

  • Cloud-hosted PostgreSQL database
  • Stores users, events, and registrations
  • Configured via PostgreSQL connection in Laravel

Make.com

  • Email automation workflows
  • Sends event confirmation/cancellation/reminder notifications
  • Triggered from registration and event actions

PSGC API

  • Provides official barangay/location data
  • Used for region → province → city → barangay selection
  • Public API source: https://psgc.gitlab.io/api/

Integration Flow

The system runs as a connected flow: users interact through the Laravel + Inertia frontend, API requests are processed in Laravel, and data is stored in Supabase. Stripe handles payments and returns webhook events to Laravel, Pusher broadcasts real-time slot updates, Make.com sends automated notifications, and PSGC API provides geographic reference data for barangay setup.

Integration Guidelines

Required tools/services:

  • Laragon (Apache + PHP environment)
  • Node.js and npm
  • Stripe CLI
  • Laravel queue worker

Commands:

  • npm run dev
  • ./stripe listen --forward-to http://barangay-events.test/api/webhooks/stripe
  • php artisan queue:listen
  • php artisan test
